# v2026.319.0

> Released: 2026-03-19

## Highlights

- **Shipping pipeline** — GHCR-based image builds, faster VPS deploys, and release workflows decoupled from every `master` push; `npm-canary` / `npm-stable` environments with `NPM_TOKEN` and fast-fail `npm whoami`. ([#29](https://github.com/Viraforge/paperclip/pull/29), [#36](https://github.com/Viraforge/paperclip/pull/36), [#37](https://github.com/Viraforge/paperclip/pull/37))
- **npm scope & CLI package** — Publish scope is `@paperclipai`; CLI ship target is `@paperclipai/cli` (binary `paperclipai` unchanged). ([#38](https://github.com/Viraforge/paperclip/pull/38), [#39](https://github.com/Viraforge/paperclip/pull/39), [#41](https://github.com/Viraforge/paperclip/pull/41))
- **Runtime stability** — Heartbeat queue/subprocess hardening, OOM mitigation, and raised VPS memory cap for the server container. ([#34](https://github.com/Viraforge/paperclip/pull/34), [#35](https://github.com/Viraforge/paperclip/pull/35))
- **Issues / checkout** — `release()` clears execution lock metadata so issues are not stuck after failed runs. ([#15](https://github.com/Viraforge/paperclip/pull/15), follow-up test coverage on `master`)
- **CI & governance** — Branch protection audit uses admin PAT where needed; workflow policy / operational docs for deploy, lockfile refresh, and npm. ([#48](https://github.com/Viraforge/paperclip/pull/48), [#44](https://github.com/Viraforge/paperclip/pull/44))
- **Agents** — `gh` in production images (wrapped config); Claude local OpenRouter billing notes and workflow policy documentation. ([#32](https://github.com/Viraforge/paperclip/pull/32), [#46](https://github.com/Viraforge/paperclip/pull/46))

## Improvements

- Longer npm publish propagation window for verify retries ([#40](https://github.com/Viraforge/paperclip/pull/40))
- Lockfile refresh landed via automation ([#43](https://github.com/Viraforge/paperclip/pull/43))
- Editor: pasted URLs no longer auto-convert to markdown links ([#33](https://github.com/Viraforge/paperclip/pull/33))

## Upgrade notes

- Prefer **`npx @paperclipai/cli@latest`** (stable) or **`@canary`** per [`doc/RELEASING.md`](../doc/RELEASING.md).
- Deploy the app image with **Deploy Vultr** when `master` should run in production; merge alone does not deploy.
